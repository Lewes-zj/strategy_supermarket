# 策略详情页 - 每日持仓 (Daily Holdings) PRD

| 版本 | 日期 | 修改人 | 备注 |
| --- | --- | --- | --- |

## 1. 页面概述 (Overview)

本页面是策略的 **“终极转化收割机”**。

* **核心逻辑：** **“看得见吃不着”**。利用模糊视觉和高亮盈亏，向用户展示“现在手里正拿着赚钱的牌”，但通过遮罩强制阻断代码查看，迫使产生强烈的解锁冲动。
* **安全原则：** **零历史数据**。本页面仅展示“当前（Today）”的持仓快照。任何已完结的历史数据均移至“交易记录”页，彻底阻断通过历史持仓推算当前持仓的逻辑漏洞。

---

## 2. 页面布局架构 (Layout)

页面采用 **“堆叠层 (Stack Layer)”** 布局，分为两层：

1. **底层 (Background Layer):** 高斯模糊处理的持仓列表（视觉诱饵）。
2. **顶层 (Foreground Layer):** 渐变遮罩 + 核心转化 CTA 组件（行动触发）。

---

## 3. UI 详细规范 (Detailed UI Specs)

### 3.1 列表表头 (Table Header)

* **样式：** 深色背景，灰色文字 (`#8C8C8C`)，字号 12px。
* **列定义：**
* `代码/名称` (Symbol/Name)
* `行业板块` (Sector)
* `方向` (Direction)
* `持仓天数` (Days Held)
* `仓位占比` (Weight)
* `累计浮盈` (Floating P&L) —— *视觉重心*



### 3.2 底层列表内容 (The Lure List)

表格需展示 **5-8 行** 数据（即使真实持仓只有 4 只，也可以通过后端补齐或视觉拉长，确保列表铺满屏幕，营造“货很足”的感觉）。

| 字段 (Column) | UI 样式规范 | 数据逻辑 |
| --- | --- | --- |
| **代码/名称** | **Icon:** `🔒` (金黄色锁图标 #FADB14)。 <br>

<br> **Text:** 高斯模糊矩形 (Blur 6px)。 <br>

<br> *视觉上要能隐约看出是文字，但绝对不可辨认。* | **防泄漏核心：** 前端只渲染模糊占位符，严禁传输真实代码。 |
| **行业板块** | **Text:** 白色高亮，清晰可见 (如 `消费电子`, `光伏设备`)。 | **诱导点：** 公开行业，让用户觉得策略在追热点。 |
| **方向** | **Tag:** 红色 `做多` (Long) / 绿色 `做空` (Short)。 | 清晰展示策略方向。 |
| **持仓天数** | **Text:** 白色数字 (如 `12天`)。 | 展示持仓周期。 |
| **仓位占比** | **Text:** 灰色数字 (如 `25%`)。 | 展示重仓/轻仓逻辑。 |
| **累计浮盈** | **Text:** **高亮红/绿字** (如 `+14.5%`)。 <br>

<br> *即使所在行被模糊/遮挡，颜色的饱和度必须穿透遮罩，隐约可见。* | **贪婪诱饵：** 让用户感觉到红彤彤的涨幅就在眼前。 |

### 3.3 顶层转化遮罩 (The Conversion Overlay)

这是一个覆盖在表格下半部分（从第 3-4 行开始）的容器。

* **背景 (Background):**
* **线性渐变 (Linear Gradient):** `Top (Transparent)` -> `40% (#141820 95% Opacity)` -> `Bottom (#141820 100% Opacity)`。
* **效果：** 列表的上半部分清晰可见（诱惑），下半部分逐渐消失在黑暗中（神秘感），无缝过渡。



#### 核心 CTA 组件 (Call To Action Group)

位于遮罩层的正中央。

1. **总浮盈大数字 (The Hero Number):**
* **Content:** `+22.3%` (动态计算的当前持仓加权总浮盈)。
* **Style:** **DIN Condensed** 字体，**64px** 超大字号，**暗红色 (#D14646)**，带有微弱的发光效果。
* *心理暗示：你只要解锁，这 22.3% 的收益逻辑就是你的。*


2. **主标题 (Headline):**
* **Content:** `当前持仓 4 只，策略实盘运行中`。
* **Style:** 18px 加粗白字。


3. **解锁按钮 (Primary Button):**
* **Content:** `🔒 立即解锁` (或 `立即解锁全部代码`)。
* **Style:** 品牌蓝 (`#1890FF`) 实心按钮，高度 48px，圆角 6px，全宽或宽版设计 (Width 240px+)。
* **Interaction:** 悬停时亮度增加，并有轻微上浮阴影。


4. **信任背书 (Trust Badge):**
* **Content:** `🛡️ 7天无理由退款`。
* **Style:** 12px 灰色文字，位于按钮正下方，图标需清晰。

## 4. 交互逻辑 (Interaction Logic)

### 4.1 点击阻断 (Click Blocking)

* **触发区域：**
* 点击列表中的 `🔒` 图标。
* 点击列表中的模糊行。
* 点击蓝色的 `立即解锁` 按钮。


* **响应动作：**
* **全部统一** 弹出 **“权益订阅弹窗 (Subscription Modal)”**。



### 4.2 弹窗传参 (Modal Context)

* 当从本页面触发弹窗时，弹窗的默认选中项应为 **“单策略订阅 (¥299)”**。
* 弹窗标题动态调整为：“解锁 Alpha Trend Strategy 的 **4只** 实时持仓”。

## 5. 数据安全与后端逻辑 (Security & Backend)

**这是本页面开发的最高优先级：**

1. **API 脱敏 (Data Masking):**
* 后端接口 `/api/strategy/holdings` 在检测到用户 `is_subscribed = false` 时：
* **字段 `symbol_code`:** 必须返回 `null` 或 `"HIDDEN"`。
* **字段 `symbol_name`:** 必须返回 `null` 或 `"HIDDEN"`。
* **字段 `current_price`:** 必须返回 `null`。
* **前端渲染：** 前端根据 `null` 值自动渲染模糊块和锁图标。**绝不能**把真实数据传给前端再用 CSS 隐藏（防止 F12 抓包泄露）。


2. **真假混合 (Real Sector, Fake ID):**
* `sector_name` (行业) 和 `pnl_percentage` (盈亏比) **必须返回真实数据**。这是为了保证“行业”和“收益”看起来是合理的，经得起推敲。


3. **总盈亏计算:**
* 遮罩层上的 `+22.3%` 必须由后端计算准确返回。公式：`Sum(Weight * P&L) for all current holdings`。

## 6. 总结 (Summary)

每日持仓 Tab 的设计放弃了传统的信息展示功能，转而完全服务于 **“情绪营销”**。
通过 `image_bc043c.png` 的视觉方案，我们移除了所有可能导致泄密的“历史数据”，仅保留“模糊的现在”和“诱人的浮盈”。这是一个逻辑闭环的商业化页面，旨在利用用户对未知收益的渴望完成最后一步转化。
