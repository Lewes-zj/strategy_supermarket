## 1. 产品概述 (Overview)

本弹窗是全站流量变现的 **“收银台”**。

* **触发场景：** 用户在“交易记录”点击解锁按钮、在“每日持仓”点击解锁遮罩、在“信号控制台”点击订阅按钮时触发。
* **核心策略：** **锚定效应 (Anchoring Effect)**。
* 左侧 **¥0 (免费版)** 作为基准，确立“只能看历史”的痛点。
* 右侧 **¥699 (PRO版)** 作为高价锚点，确立“全站通”的高端定位。
* 中间 **¥299 (本策略)** 作为**“最具性价比”**的推荐项，承接 80% 的转化流量。

## 3. 交互逻辑 (Interaction Logic)

### 3.1 默认选中状态

* 弹窗打开时，**中间的“本策略订阅”卡片** 默认处于高亮/选中状态。
* 用户的视觉焦点应第一时间落在蓝色的 `立即订阅` 按钮上。

### 3.2 动态内容渲染 (Dynamic Context)

后端 API 需根据触发来源，动态填充弹窗内容：

* **Source = Strategy Page (详情页):**
* Title: `解锁 [Strategy_Name] 核心权益`.
* Plus Card Price: 读取该策略的单价 (通常 ¥299).


* **Source = Global Nav (会员中心):**
* Title: `升级 PRO 会员，解锁全站策略`.
* **默认选中右侧 PRO 卡片**。



### 3.3 支付流程 (Payment Flow)

1. 用户点击 `立即订阅`。
2. 按钮变为 `Loading...` 状态。
3. 调用支付网关 (Stripe / WeChat Pay)。
4. **支付成功：**
* 弹窗关闭。
* 页面自动刷新。
* **效果：** 列表中的 `****` 瞬间变为真实代码，锁图标消失，Toast 提示“解锁成功，实时信号已开启”。



### 3.4 异常处理

* **网络错误：** Toast 提示“连接支付网关失败，请重试”。
* **重复订阅：** 若用户已有该权限，按钮文案显示为 `已订阅` 且不可点，或引导 `续费`。
